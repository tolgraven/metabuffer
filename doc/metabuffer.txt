==============================================================================
INTRODUCTION\t\t\t\t*metabuffer-introduction*

metabuffer provides an interactive buffer-line filter with matcher/case/syntax
cycling and optional line push-back to source.

==============================================================================
USAGE\t\t\t\t\t*metabuffer-usage*

Run |:Meta| to start filtering current buffer lines.

==============================================================================
COMMANDS\t\t\t\t*metabuffer-commands*

\t\t\t\t\t*:Meta*
\t\t\t\t\t*:MetaCursorWord*
:Meta [{query}]
:MetaCursorWord
\tStart metabuffer from current buffer.

\t\t\t\t\t*:MetaResume*
\t\t\t\t\t*:MetaResumeCursorWord*
:MetaResume [{query}]
:MetaResumeCursorWord
\tResume latest metabuffer state for the current source buffer.

\t\t\t\t\t*:MetaSync*
:MetaSync [{query}]
\tProgrammatically update active query.

\t\t\t\t\t*:MetaPush*
:MetaPush
\tPush edited visible metabuffer lines back to source line locations.

SETUP OPTIONS\t\t\t\t*metabuffer-setup-options*

Animation setup options:

\tui.animation.enabled
\t\tMaster enable/disable switch for Meta animations.

\tui.animation.backend
\t\tGlobal animation backend. "mini" is the default and uses
\t\tmini.animate where supported. "native" keeps Meta's built-in
\t\tfallback behavior.

\tui.animation.time_scale
\t\tMaster speed multiplier. 1.0 is normal speed, 0.5 is twice as
\t\tfast, 2.0 is half speed.

\tui.animation.prompt.enabled
\tui.animation.prompt.time_scale
\t\tControls prompt split slide animation.

\tui.animation.preview.enabled
\tui.animation.preview.time_scale
\t\tControls preview window slide animation.

\tui.animation.info.enabled
\tui.animation.info.time_scale
\t\tControls info window fade/slide animation.

\tui.animation.loading.enabled
\tui.animation.loading.time_scale
\t\tControls animated loading word cadence in the results statusline.

\tui.animation.scroll.enabled
\tui.animation.scroll.time_scale
\t\tControls animated half/page scrolling in the results window.

\tPer-animation backend keys are still accepted as compatibility
\toverrides, but ui.animation.backend is the intended control surface.

\tui.animation.loading_indicator
\t\tIndependent toggle for showing the loading indicator at all.

\tBase durations stay internal. Public configuration uses enable flags and
\ttime scales only.

PROMPT KEYS\t\t\t\t*metabuffer-prompt-keys*

Default prompt mappings:

Accept/cancel:
\t<CR>\tAccept current selection, or apply selected history-browser item.
\t<Esc>\tNormal mode: cancel. If history browser is open, closes browser first.

Navigation:
\t<C-p>, <C-n>, <C-j>, <C-k>\tMove selection
\t<Up>, <Down>\t\t\tHistory browse or move selection

Shell/history shorthands (insert + normal):
\t!!\tInsert latest prompt history entry inline
\t!$\tInsert last token from latest prompt history entry
\t!^!\tInsert latest prompt history entry except first token

Insert-mode editing:
\t<C-a>\tMove cursor to line start
\t<C-e>\tMove cursor to line end
\t<C-u>\tKill from line start to cursor
\t<C-k>\tKill from cursor to line end
\t<C-y>\tYank killed prompt text
\t<S-CR>\tInsert newline at cursor in prompt buffer

Prompt token operators:
\t<LocalLeader>1, <LocalLeader>!\tToggle leading ! on token under cursor
\t<C-r>\t\t\t\tOpen floating history searchback
\t<LocalLeader>h\t\t\tMerge persisted history into current session cache

Main/results buffer:
\t!\tAppend !<cword> to prompt query

PROMPT DIRECTIVES\t\t\t*metabuffer-prompt-directives*

Most directives are consumed from prompt text after applying:

All #toggles
	Options:
	#escape / #e	Disable prefiltering.
	#exp / #ex {expander}	Set the active expansion mode.
	#history / #h	Merge persisted history into the current session.
	#lazy / #l	Enable lazy project loading.
		Disable with #-lazy, #nolazy, #-l.
	#prefilter / #p	Enable project lazy prefiltering.
		Disable with #-prefilter, #noprefilter, #-p.
	#save / #s {tag}	Save the current prompt under a tag.
	##	Open the saved-prompt browser.
	##{tag}	Restore a saved prompt inline.
	Scope:
	#binary / #b	Include binary files.
		Disable with #-binary, #nobinary, #-b.
	#deps / #d	Include dependency and vendor paths.
		Disable with #-deps, #nodeps, #-d.
	#hidden / #hi	Include hidden paths.
		Disable with #-hidden, #nohidden, #-hi.
	#ignored / #i	Include ignored paths.
		Disable with #-ignored, #noignored, #-i.
	Transforms:
	#b64 / #b6	Decode obvious base64 text before display and filtering.
		Disable with #-b64, #nob64, #-b6.
	#bplist / #bp	Pretty-print binary plist files.
		Disable with #-bplist, #nobplist, #-bp.
	#css / #c	Pretty-print minified CSS lines.
		Disable with #-css, #nocss, #-c.
	#hex / #he	Render binary files through hex view.
		Disable with #-hex, #nohex, #-he.
	#json / #j	Pretty-print minified JSON lines.
		Disable with #-json, #nojson, #-j.
	#strings / #st	Extract printable strings from binary files.
		Disable with #-strings, #nostrings, #-st.
	#xml / #x	Pretty-print minified XML lines.
		Disable with #-xml, #noxml, #-x.
	Sources:
	#file / #f {token}	Switch to file-entry source filtering.
		Disable with #-file, #nofile, #-f.
	#lgrep / #lg {query}	Switch the source set to lgrep semantic search hits.
	#lgrep:d / #lg:d {symbol}	Switch the source set to lgrep definitions for a symbol.
	#lgrep:u / #lg:u {symbol}	Switch the source set to lgrep usages for a symbol.
End #toggles

\tRelated setup options:
\t\tdefault_include_lgrep
\t\tlgrep_bin
\t\tlgrep_limit
\t\tlgrep_debounce_ms

\tCustom transforms:
\t\tDefine them under `options.custom.transforms`.
\t\tInvoke them as `#transform:name`.
\t\tShort aliases are derived automatically at runtime; completion/help
\t\tshows the resolved alias.
\t\tEach transform accepts:
\t\t- `from`: required shell command (argv list or shell string)
\t\t- `to`: optional reverse shell command for writeback
\t\t- `scope`: `line` or `file` (default `line`)
\t\t- `applies_to`: `text`, `binary`, or `all` (default `text`)
\t\t- `filetypes`: optional list of detected filetypes to limit application
\t\t- `filetype_commands`: optional per-filetype command map overriding `from`/`to`
\t\t- `enabled`: optional default state
\t\t- `doc`: help text for completion/popup docs
\t\tExample:
\t\t\tdecompyle = {
\t\t\t  filetype_commands = {
\t\t\t    python = {
\t\t\t      from = { "decompyle3", "-" },
\t\t\t    },
\t\t\t  },
\t\t\t  scope = "file",
\t\t\t  applies_to = "binary",
\t\t\t  filetypes = { "python" },
\t\t\t  doc = "Decompile Python bytecode files.",
\t\t\t}

PROMPT BEHAVIOR\t\t\t*metabuffer-prompt-behavior*

- `all` matcher supports token negation via `!token`.
- `^` and `$` anchors work per token.
- Regex-like tokens (for example containing `(`, `[`, `\\w`, `+`, `*`, `?`, `|`)
  are treated as regex for that token only when valid.
- Invalid/incomplete regex tokens fall back to literal matching.
- Negated plain word tokens prefer word-boundary exclusion.
- Prompt highlights:
\t- Negated token (`!word`) -> error-style highlight
\t- Anchors (`^`, `$`) -> anchor highlight
\t- Regex token -> regex highlight (underlined)

- History browser behavior:
\t- Typing stays in prompt; floating browser updates from prompt text
\t- <CR> writes selected browser entry into prompt text
\t- <Esc> closes browser first, then Meta on next cancel

- Persistence:
\t- Prompt history and saved tags are stored at:
\t  `stdpath("data")/metabuffer_prompt_history.json`
